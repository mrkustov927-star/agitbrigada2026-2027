# Фронтовая агитбригада — командный штаб

Внутренний веб-дашборд команды грантового проекта. Система контролирует календарный план, участников 14–35 лет, публикации и просмотры, бюджет, подтверждающие документы, качественные результаты и готовность приложений итогового отчёта Росмолодёжь.Гранты.

## Возможности

- общая база данных и синхронизация между устройствами;
- вход по e-mail и паролю через Supabase Auth;
- роли: руководитель, координатор, финансы, медиа, организатор, наставник, просмотр;
- серверное хранение рабочего состояния проекта;
- закрытое файловое хранилище подтверждающих документов;
- автоматические предупреждения по срокам и показателям;
- экспорт реестров и черновика итогового отчёта;
- локальный резервный режим, если Supabase не настроен.

## Запуск локально

```bash
npm install
cp .env.example .env.local
npm run dev
```

Проект уже содержит безопасные публичные параметры подключения к рабочему Supabase-проекту. Для другого окружения их можно переопределить через `.env.local`:

```env
VITE_SUPABASE_URL=https://PROJECT_REF.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=ваш_publishable_key
VITE_PROJECT_CODE=AGITBRIGADA-2026-2027
```

`service_role`, secret key и пароль базы нельзя помещать во frontend или в репозиторий.

## Настройка Supabase

В SQL Editor последовательно выполнить:

1. `supabase_schema.sql`
2. `supabase_seed.sql`
3. `supabase_shared_state.sql`

Первые два файла создают нормализованную проектную базу и исходные данные. Третий файл добавляет совместимый слой синхронизации автономного MVP и включает Realtime.

## Первый вход

1. Разверните сайт и откройте его.
2. Нажмите «Создать аккаунт».
3. Первый зарегистрированный пользователь автоматически получает роль `owner`, если в проекте ещё нет активных участников.
4. Остальные пользователи сначала регистрируются, затем руководитель добавляет их e-mail в команду.

## Развёртывание на GitHub Pages

В репозитории есть workflow `.github/workflows/deploy-pages.yml`. После слияния изменений в `main` откройте `Settings → Pages` и выберите источник `GitHub Actions`. Сайт будет собран и опубликован автоматически.

Предполагаемый адрес:

```text
https://mrkustov927-star.github.io/agitbrigada2026-2027/
```

В Supabase добавьте этот адрес в `Authentication → URL Configuration` как `Site URL` и `Redirect URL`.

## Развёртывание на Vercel

- Framework preset: `Vite`
- Build command: `npm run build`
- Output directory: `dist`
- Environment variables: `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`, `VITE_PROJECT_CODE`

После изменения переменных выполните новый deployment.
